# 용어집 (Glossary)

> Canton Network 개념에 대한 권위 있는 용어 레퍼런스

> 참고: 표제어는 "한국어 (English)" 형식으로 표기하며, **관련** 항목은 함께 보면 좋은 용어다.

## A

### 활성 컨트랙트 집합 (Active Contract Set, ACS)

참여자 노드에서 자신이 호스팅하는 <abbr class="gloss" title="Canton에서 권한과 데이터 가시성의 주체가 되는 식별 가능한 참여 주체">파티</abbr>에 대해 현재 활성(보관되지 않은) 상태인 <abbr class="gloss" title="원장에 기록되는 불변 데이터 단위. 상태 변경은 새 컨트랙트 생성으로 표현됨">컨트랙트</abbr>의 집합. ACS는 그 참여자 관점에서의 현재 상태를 나타낸다.

**관련**: 컨트랙트, 보관(Archive), 참여자 노드

### Admin API

파티 관리, 패키지 업로드, 노드 구성 같은 운영 작업을 위해 Canton 노드가 노출하는 관리용 API. 애플리케이션이 쓰는 Ledger API와 분리되어 있다.

**관련**: Ledger API, 참여자 노드

### 보관 (Archive)

컨트랙트를 소비/삭제하는 동작. 보관된 컨트랙트는 더 이상 행위 대상이 될 수 없다. <abbr class="gloss" title="다자간 워크플로를 위해 설계된 Canton의 스마트 컨트랙트 언어">Daml</abbr>에서 소비형 <abbr class="gloss" title="컨트랙트에서 수행 가능한 동작(권한이 부여된 당사자만 실행 가능)">초이스</abbr>를 실행하면 컨트랙트가 보관된다.

**관련**: 컨트랙트, 초이스, 활성 컨트랙트 집합

## C

### Canton

프라이버시 보존 분산 원장 기술을 구현하는 블록체인 프로토콜과 소프트웨어. Canton은 기반 기술이고, Canton Network은 그것을 실행하는 퍼블릭 네트워크다.

**관련**: Canton Network, <abbr class="gloss" title="상태를 저장하지 않고 트랜잭션 합의·순서를 조율하는 Canton 구성요소">동기화자</abbr>, 참여자 노드

### Canton Coin (CC)

<abbr class="gloss" title="슈퍼 밸리데이터들이 공동 운영하는 Canton의 퍼블릭 조율(합의) 계층">글로벌 동기화자</abbr>의 네이티브 유틸리티 토큰으로, 트랜잭션 수수료(트래픽)와 <abbr class="gloss" title="파티를 호스팅하고 그 파티의 컨트랙트 데이터를 저장하는 참여자 노드">밸리데이터</abbr> 보상에 쓰인다. 일부 맥락에서 과거 "Amulet"로 불렸다.

**관련**: 트래픽, 글로벌 동기화자, Splice

### Canton 콘솔 (Canton Console)

Canton 노드를 관리하는 대화형 명령줄 인터페이스. 파티 관리, 패키지 배포, 디버깅 같은 운영 작업을 위한 Scala 기반 스크립팅을 제공한다.

**관련**: Admin API, 참여자 노드

### Canton Network

Canton 프로토콜 위에서 실행되는 퍼블릭 블록체인 네트워크로, 글로벌 동기화자가 조율하고 <abbr class="gloss" title="글로벌 동기화자를 운영하고 네트워크 거버넌스에 참여하는 노드">슈퍼 밸리데이터</abbr>가 운영한다.

**관련**: Canton, 글로벌 동기화자, 슈퍼 밸리데이터

### 초이스 (Choice)

Daml 컨트랙트에서 실행할 수 있는 동작. 초이스는 특정 파티가 통제하며 다음일 수 있다:

* **소비형(Consuming)**: 실행 시 컨트랙트를 보관
* **비소비형(Non-consuming)**: 컨트랙트를 활성으로 둠

**관련**: 컨트랙트, <abbr class="gloss" title="컨트랙트의 구조와 규칙(권한·초이스)을 정의하는 Daml 청사진">템플릿</abbr>, 컨트롤러

### 확인 참여자 노드 (Confirming Participant Node, CPN)

자신이 호스팅하는 파티를 대신해 트랜잭션을 확인할 권한이 있는 참여자 노드. 트랜잭션 확인 프로토콜의 일부.

**관련**: 참여자 노드, 트랜잭션

### 컨트랙트 (Contract)

원장에 있는 템플릿의 인스턴스. 컨트랙트는 불변이다 — 변경은 새 컨트랙트를 생성하고 옛것을 보관한다. 각 컨트랙트는 고유한 컨트랙트 ID를 갖는다.

**관련**: 템플릿, 보관, 컨트랙트 ID

### 컨트랙트 ID (Contract ID)

원장의 컨트랙트 인스턴스에 대한 고유 식별자. 초이스와 트랜잭션에서 특정 컨트랙트를 참조하는 데 쓰인다.

**관련**: 컨트랙트, 컨트랙트 키

### 컨트랙트 키 (Contract Key)

컨트랙트 ID를 모르고도 컨트랙트를 조회하기 위한 고유 식별자. 키는 템플릿 범위 내에서 고유해야 하며 프라이버시 함의를 갖는다 — 키 구조가 정보를 드러낼 수 있다.

**관련**: 컨트랙트, 컨트랙트 ID, 프라이버시

### 컨트롤러 (Controller)

컨트랙트의 특정 초이스를 실행할 권한이 있는 파티. 템플릿 정의에서 `controller` 키워드로 선언된다.

**관련**: 초이스, 서명자, 관찰자

## D

### Daml

Canton에서 쓰이는 스마트 컨트랙트 언어. 권한과 프라이버시를 내장한, 다자간 워크플로를 위해 설계된 함수형 프로그래밍 언어. Daml-LF로 컴파일된다.

**관련**: 템플릿, 초이스, Daml-LF

### Daml-LF

Daml Ledger Format — Daml 컨트랙트의 컴파일된 바이트코드 형식. Canton 참여자 노드에서 실행되는 중간 표현(intermediate representation).

**관련**: Daml, DAR

### DAR (Daml Archive)

참여자 노드에 배포할 준비가 된 컴파일된 Daml 코드 패키지(템플릿, 데이터 타입 등). `daml build`로 생성된다.

**관련**: Daml, Daml-LF, 패키지

### DevNet

Canton Network의 개발/테스트 환경. VPN 접근과 슈퍼 밸리데이터 후원이 필요하다. 포셋의 테스트 <abbr class="gloss" title="트랜잭션 수수료와 밸리데이터 보상에 쓰이는 네이티브 유틸리티 토큰(CC)">Canton Coin</abbr>을 사용한다.

**관련**: TestNet, MainNet, LocalNet

### 디벌전스 (Divulgence)

원래 이해관계자가 아니었으나 트랜잭션 조합을 통해 관여하게 된 파티에게 컨트랙트 정보가 자동 공개되는 것. 트랜잭션에서 컨트랙트가 페치(fetch)될 때 일어난다.

**관련**: 프라이버시, 관찰자, <abbr class="gloss" title="한 트랜잭션을 &quot;뷰&quot;로 분해해, 각 파티가 자신과 관련된 부분만 보도록 하는 Canton의 핵심 프라이버시 방식">부분 트랜잭션 프라이버시</abbr>

### DSO (탈중앙화 동기화자 운영자, Decentralized Synchronizer Operator)

탈중앙화 동기화자에서 노드를 적극적으로 운영하는 슈퍼 밸리데이터의 집합. 거버넌스 결정을 내리고 인프라를 운영한다.

**관련**: 슈퍼 밸리데이터, 글로벌 동기화자

### DSO 파티 (DSO Party)

공동 동작을 위한 서명을 모으고 DSO를 위한 동기화 인프라를 유지하는 데 쓰이는 특수 파티 구성.

**관련**: DSO, 파티

## E

### 외부 파티 (External Party)

참여자 노드 외부에 자신의 서명 키를 보유해, 트랜잭션에 명시적 서명을 요구하는 파티. Ethereum의 외부 소유 계정(EOA)과 유사하다. 더 많은 통제를 주지만 외부 서명 흐름이 필요하다.

**관련**: 로컬 파티, 파티

## G

### 글로벌 동기화자 (Global Synchronizer)

슈퍼 밸리데이터가 운영하는 퍼블릭·탈중앙화 동기화자로, Canton Network의 백본이다. Canton Foundation이 거버넌스한다.

**관련**: 동기화자, 슈퍼 밸리데이터, Canton Foundation

### Canton Foundation (CF)

글로벌 동기화자를 거버넌스하는 Linux Foundation 산하의 독립적 비영리 기구. 네트워크 정책을 설정하고, 업그레이드를 조율하고, 슈퍼 밸리데이터 참여를 감독한다.

**관련**: 글로벌 동기화자, 슈퍼 밸리데이터

### gRPC

Ledger API의 주된 프로토콜로, 직렬화에 Protocol Buffers를 사용한다. JSON API보다 성능이 높다.

**관련**: Ledger API, JSON Ledger API

## H

### 호스팅 (Hosting)

참여자 노드와 그것이 호스팅하는 파티 사이의 관계. 참여자는 호스팅 파티의 데이터를 저장하고 그들을 대신해 행위한다.

**관련**: 참여자 노드, 파티

## I

### 인터페이스 (Interface)

여러 템플릿이 구현할 수 있는 초이스 집합을 정의하는 Daml 구성. 다형적(polymorphic) 컨트랙트 처리를 가능하게 한다.

**관련**: 템플릿, 초이스

## J

### JSON Ledger API

Ledger API의 HTTP/JSON 변형. gRPC보다 사용하기 쉽지만 약간의 성능 오버헤드가 있다. 브라우저 기반 애플리케이션에 자주 쓰인다.

**관련**: Ledger API, gRPC

### JWT (JSON Web Token)

Canton API가 쓰는 인증 메커니즘. 토큰은 파티 신원과 권한을 인코딩한다.

**관련**: Ledger API, Admin API

## L

### Ledger API

커맨드 제출과 원장 데이터 읽기를 위해 참여자 노드가 노출하는 주된 API. gRPC와 JSON 형식으로 제공된다. 애플리케이션이 원장과 상호작용하는 데 쓴다.

**관련**: gRPC, JSON Ledger API, Admin API

### 로컬 파티 (Local Party)

호스팅 참여자 노드가 키를 보유해, 노드가 파티를 대신해 서명할 수 있는 파티. 외부 파티보다 단순하지만 밸리데이터에게 완전한 통제를 준다.

**관련**: 외부 파티, 파티, 참여자 노드

### LocalNet

Canton 구성 요소를 자기 머신에서 실행하는 로컬 개발 환경. 외부 네트워크가 필요 없으며 시뮬레이션된 Canton Coin을 사용한다.

**관련**: DevNet, TestNet, MainNet

## M

### MainNet

Canton Network의 프로덕션 환경. 경제 가치가 있는 실제 Canton Coin을 사용한다. 완전한 온보딩 절차가 필요하다.

**관련**: DevNet, TestNet, LocalNet

### 미디에이터 (Mediator)

참여자로부터 확인을 수집하고 트랜잭션 결과(커밋 또는 거부)를 선언함으로써 트랜잭션 확인 프로토콜을 촉진하는 동기화자 구성 요소.

**관련**: 동기화자, 시퀀서, 트랜잭션

## N

### 네임스페이스 (Namespace)

키 지문(fingerprint)으로 정의되는 암호학적 범위. 파티와 다른 <abbr class="gloss" title="어떤 노드·파티·키가 네트워크에 참여하는지를 정의하는 구성 정보">토폴로지</abbr> 엔티티가 네임스페이스 안에 존재한다. 그 네임스페이스의 엔티티에 대한 토폴로지를 누가 수정할 수 있는지 통제한다.

**관련**: 파티, 토폴로지 트랜잭션

## O

### 관찰자 (Observer)

컨트랙트를 볼 수 있지만 (컨트롤러로도 선언되지 않으면) 초이스를 실행할 수 없는 파티. 템플릿에서 `observer` 키워드로 명시적으로 선언된다.

**관련**: 서명자, 컨트롤러, 프라이버시

### 오프셋 (Offset)

원장 트랜잭션 스트림에서의 위치. Ledger API에서 읽을 때 페이지네이션과 재개에 쓰인다.

**관련**: Ledger API, 트랜잭션

## P

### 패키지 (Package)

참여자 노드에 배포되는 컴파일된 Daml 모듈(템플릿, 데이터 타입)의 모음. DAR 파일로 패키징된다.

**관련**: DAR, 템플릿, 베팅(Vetting)

### 참여자 노드 (Participant Node)

파티를 호스팅하고, 그들의 컨트랙트 데이터를 저장하고, 트랜잭션을 검증하고, Canton 프로토콜에 참여하는 노드. Canton Network 맥락에서 "밸리데이터 노드"라고도 한다.

**관련**: 밸리데이터 노드, 파티, 호스팅

### 파티 (Party)

Canton의 온-원장 신원으로, 주소나 계정과 유사하다. `name::fingerprint` 형식이며, 지문은 파티의 공개키에서 파생된다.

**관련**: 로컬 파티, 외부 파티, 서명자

### PQS (참여자 쿼리 스토어, Participant Query Store)

성능이 중요한 애플리케이션을 위해 원장 데이터의 SQL 기반 조회를 제공하는 선택적 구성 요소. 원장 상태와 동기화된 PostgreSQL 데이터베이스를 유지한다.

**관련**: Ledger API, 활성 컨트랙트 집합

### 프루닝 (Pruning)

성장을 관리하기 위해 참여자 저장소에서 더 이상 필요 없는 오래된 데이터를 제거하는 과정. <abbr class="gloss" title="더 이상 필요 없는 과거 원장 데이터를 정리해 저장공간을 줄이는 작업">프루닝</abbr>된 데이터는 더 이상 조회할 수 없지만 원장은 유효하게 유지된다.

**관련**: 참여자 노드, 활성 컨트랙트 집합

## S

### 시퀀서 (Sequencer)

참여자로부터 온 메시지를 정렬하고 수신자에게 분배하는 동기화자 구성 요소. 복호화된 메시지 내용을 보지 않고 전체 순서를 제공한다.

**관련**: 동기화자, 미디에이터

### 서명자 (Signatory)

컨트랙트 생성을 승인해야 하며 컨트랙트 의무에 책임을 지는 파티. 서명자는 항상 컨트랙트와 그 위의 모든 동작을 본다.

**관련**: 관찰자, 컨트롤러, 템플릿

### Splice

탈중앙화 Canton 동기화자를 운영·재정지원·거버넌스하는 인프라를 제공하는 오픈소스 프로젝트(Hyperledger Labs 산하). Canton Coin과 관련 애플리케이션을 구현한다.

**관련**: Canton Coin, 글로벌 동기화자, DSO

### 이해관계자 (Stakeholder)

컨트랙트에 지분을 가진 파티 — 서명자 또는 관찰자. 이해관계자는 자신이 지분을 가진 컨트랙트의 트랜잭션 뷰를 받는다.

**관련**: 서명자, 관찰자, 뷰

### 부분 트랜잭션 프라이버시 (Sub-transaction Privacy)

Canton의 핵심 프라이버시 기능 — 트랜잭션을 뷰로 분해해 각 파티가 자신과 관련된 부분만 보게 한다. 참여자는 권한이 없는 뷰에 대해서는 내용도 메타데이터도 보지 못한다.

**관련**: 뷰, 프라이버시, 트랜잭션

### 슈퍼 밸리데이터 (Super Validator, SV)

글로벌 동기화자의 탈중앙화 인프라에서 노드를 운영하는 주체. 슈퍼 밸리데이터는 시퀀서/미디에이터 노드를 운영하고, 거버넌스에 참여하며, 새 밸리데이터를 후원한다.

**관련**: 글로벌 동기화자, DSO, 밸리데이터 노드

### 동기화자 (Synchronizer)

트랜잭션을 정렬하고 참여자 노드 간 합의를 촉진하는 조율 계층. 시퀀서(들)와 미디에이터(들)로 구성된다. 트랜잭션 내용을 저장하지 않는다.

**관련**: 시퀀서, 미디에이터, 글로벌 동기화자

## T

### 템플릿 (Template)

컨트랙트 데이터, 서명자, 관찰자, 초이스를 명시하는 Daml 정의. 컨트랙트는 템플릿의 인스턴스다. 객체지향 프로그래밍의 클래스와 유사하다.

**관련**: 컨트랙트, 초이스, Daml

### TestNet

MainNet 배포 전 최종 검증에 쓰이는 Canton Network의 스테이징 환경. 포셋의 테스트 Canton Coin을 사용한다.

**관련**: DevNet, MainNet, LocalNet

### 토폴로지 트랜잭션 (Topology Transaction)

네트워크 토폴로지 — 파티 호스팅 관계, 키 권한, 동기화자 연결 등 — 를 수정하는 트랜잭션. Admin API와 Canton 콘솔로 관리된다.

**관련**: 네임스페이스, 파티, Admin API

### 트래픽 (Traffic)

Canton Network에서 트랜잭션 수수료를 가리키는 용어로, Canton Coin으로 지불한다. 트래픽 비용은 트랜잭션 크기와 복잡도에 좌우된다.

**관련**: Canton Coin, 트랜잭션

### 트랜잭션 (Transaction)

하나 이상의 액션(생성, 보관, 실행)으로 이루어진 원장 업데이트. 트랜잭션은 원자적이다 — 모든 액션이 성공하거나 모두 실패한다.

**관련**: 뷰, 부분 트랜잭션 프라이버시

### 트랜잭션 ID (Transaction ID)

원장의 트랜잭션에 대한 고유 식별자. 추적과 감사에 쓰인다.

**관련**: 트랜잭션, 컨트랙트 ID

## U

### Update

원장 연산(생성, 실행, 페치)을 나타내는 Daml 모나드. Daml 트랜잭션 로직의 핵심 빌딩 블록.

**관련**: Daml, 트랜잭션

### 사용자 (User)

하나 이상의 파티에 매핑되는 인증 계층(JWT)의 신원. 사용자는 Ledger API에 인증하고 자신과 연관된 파티로서 행위한다.

**관련**: 파티, JWT, Ledger API

## V

### 밸리데이터 노드 (Validator Node)

파티를 호스팅하고, 그들의 컨트랙트 데이터를 저장하고, 트랜잭션을 검증하고, Canton 프로토콜에 참여하는 노드. Canton Network 맥락에서 "참여자 노드"와 혼용된다.

**관련**: 참여자 노드, 슈퍼 밸리데이터

### 베팅 (Vetting)

참여자에서 패키지를 사용하도록 승인하는 과정. 그 패키지의 컨트랙트를 생성하려면 패키지가 베팅되어야 한다. 배포된 코드에 대한 보안 통제를 제공한다.

**관련**: 패키지, DAR, 참여자 노드

### 뷰 (View)

특정 파티 또는 파티 집합에게 보이는 트랜잭션의 일부. 부분 트랜잭션 프라이버시는 뷰를 통해 구현된다 — 각 참여자는 자신이 권한 있는 뷰만 받는다.

**관련**: 부분 트랜잭션 프라이버시, 트랜잭션, 이해관계자

## W

### 월렛 (Wallet)

Canton Coin을 관리하고 Canton Network과 상호작용하는 사용자용 애플리케이션. Splice 프로젝트가 참조 월렛 구현을 제공한다.

**관련**: Canton Coin, Splice, Wallet SDK

### Wallet SDK

월렛 기능을 애플리케이션에 통합하기 위한 SDK. 잔액 조회, 이전, 사용자 관리를 위한 API를 제공한다.

**관련**: 월렛, Splice, Ledger API
