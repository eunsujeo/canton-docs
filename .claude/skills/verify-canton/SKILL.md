# verify-canton

`translate-canton`으로 만든 한국어 위키의 정확성과 일관성을 검증한다. 규칙 전문은 루트 `CLAUDE.md`, 용어 규칙은 `wiki/glossary.md` 참고.
검증은 **수정하지 않고 보고**가 기본이다. 사용자가 "고쳐줘"라고 하면 그때 수정한다.

## 두 가지 모드
- **page**: 특정 페이지(또는 여러 페이지)를 원문과 1:1 대조하는 번역 품질 검증.
- **lint**: 위키 전체의 구조적 건강도 점검.
모드 지정이 없으면, 대상이 특정되면 page, 아니면 lint로 판단한다.

---

## 모드 1: page — 번역 품질 검증
대상 페이지마다 다음을 수행한다.

1. **원문 확보**: 페이지 frontmatter의 `source:` URL에서 `<url>.md`를 가져온다.
   - 예: `curl -s <source>.md -o /tmp/canton-orig.md`
   - `source:`가 없으면 즉시 **[출처 누락]** 결함으로 보고.
2. **번역본 읽기**: `wiki/<PATH>.md`.
3. 아래 체크리스트로 대조하고, 항목별 ✅/⚠️/❌ 판정과 근거(줄/문단 인용)를 낸다.

### 체크리스트
- **[출처]** frontmatter `source:` + 상단 `> 출처(원문)` 링크가 있고 URL이 실제 원문과 일치하는가.
- **[구조 충실도]** 원문의 모든 제목(heading)이 번역본에 빠짐없이 대응되는가. 누락/추가된 섹션이 없는가.
  - 빠른 확인: 원문과 번역본의 heading 개수 비교 → `grep -c '^#' /tmp/canton-orig.md` vs `grep -c '^#' wiki/<PATH>.md` (개발자 노트 헤더 1~2개 차이는 정상).
- **[내용 누락]** 문단·목록·표의 행이 빠지거나 임의 요약되지 않았는가. (요약은 상단 "개발자 노트"에만 허용.)
- **[코드 불변]** 코드 블록·명령어·식별자·API/타입/함수명·경로·mermaid 다이어그램이 번역되지 않고 원문 그대로인가.
- **[고유명]** Canton, Daml, Splice, Canton Coin, Global Synchronizer 등 고유명이 원문 유지됐는가.
- **[용어 일관성]** `wiki/glossary.md`의 대응표를 따랐는가. 같은 용어가 페이지 내/페이지 간에 다르게 번역되지 않았는가.
- **[표/링크]** 표 구조가 유지됐는가. 내부 링크가 규칙대로(번역됐으면 상대경로, 아니면 원문 URL) 처리됐는가. 링크 깨짐 없는가.
- **[오역/누락 의미]** 핵심 문장의 의미가 왜곡되지 않았는가 (특히 부정문, 조건문, "less ideal" 같은 대조 표현).

### 보고 형식
```
## 검증: <제목> (wiki/<PATH>.md)
- 종합: ✅ 통과 / ⚠️ 경미 / ❌ 수정 필요
- [출처] ✅ …
- [구조 충실도] ⚠️ 원문 H2 8개 vs 번역 7개 — "When to Use Canton" 누락 의심 (원문 L80)
- [용어 일관성] ❌ "validator"가 "검증자"/"밸리데이터" 혼용 (L40, L72) → glossary는 "밸리데이터"
- 권장 조치: …
```

## 모드 2: lint — 위키 건강도 점검
전체 위키를 대상으로 아래를 실행해 결함 목록을 만든다.

1. **상태 불일치 (sources.md ↔ 실제 파일)**
   - `sources.md`에서 ☑인데 파일이 없는 경우, ☐인데 파일이 존재하는 경우를 찾는다.
   - 디스크의 번역 파일 목록: `find wiki -name '*.md' -not -name index.md -not -name sources.md -not -name glossary.md -not -name log.md`
2. **출처 누락**: 모든 번역 페이지에 `source:` frontmatter가 있는지.
   - `grep -L 'source:' $(find wiki/overview wiki/appdev wiki/global-synchronizer wiki/integrations -name '*.md' 2>/dev/null)` → 출력된 파일이 결함.
3. **인덱스 누락**: 번역 완료(☑) 페이지가 `wiki/index.md` 카테고리 섹션에 등재됐는지.
4. **깨진/고아 링크**: 위키 내부 상대링크가 실제 존재하는 파일을 가리키는지. 어떤 페이지에서도 링크되지 않는 고아 페이지를 찾는다.
5. **용어 불일치**: `glossary.md` 대응표의 영문 용어가 본문에서 서로 다른 한국어로 번역된 사례를 표본 점검한다.
6. **구식/모순**: 같은 개념을 다르게 설명하는 페이지 간 모순을 점검한다.
7. **로그 정합성**: 최근 ingest가 `log.md`에 기록됐는지.

### 보고
- 결함을 **심각도(❌ 수정 필요 / ⚠️ 경미)**별로 묶어 목록화하고, 각 항목에 파일 경로와 한 줄 조치 제안을 단다.
- 마지막에 진행 요약: `완료 N / 남음 M` (`grep -c '| ☑ |' wiki/sources.md`).
- 새로 만들면 좋을 페이지(자주 언급되나 없는 핵심 개념)나 채울 데이터 갭이 보이면 제안한다.

## 원칙
- 검증은 보고가 우선. 수정은 사용자 승인 후. 수정 시 `translate-canton`의 갱신 절차(sources/index/log)를 동일하게 따른다.
- 추측으로 "통과" 판정하지 않는다. 원문을 실제로 가져와 대조한 근거로만 판정한다.
